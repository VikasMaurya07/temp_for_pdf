💠 Edmonds' Blossom Algorithm

This project implements Edmonds' Blossom Algorithm (also known as the General Graph Maximum Matching Algorithm) in C++. Blossom contraction and alternating-path augmentation are the core techniques that make this possible.

Problem Statement

Given an undirected simple graph with ( N ) vertices and ( M ) edges, find the maximum matching: a set of edges such that no two edges share a common vertex, and the number of such edges is maximized.

Input Format 

N M
u0 v0
u1 v1
...
u(M-1) v(M-1)

Output Format 

X
a0 b0
a1 b1
...
a(X-1) b(X-1)

Where:

  • X = size of the maximum matching
  • Each line (ai, bi) represents one matched edge (ai < bi).

🧠 Algorithmic Intuition

🔹Matching & Augmenting Paths

  • A matching is a set of edges with no shared endpoints.
  • If there exists a path that alternates between unmatched and matched edges, starting and ending with unmatched vertices, it’s called an augmenting path.
  • Flipping edges along this path increases the matching size by 1.

🔹The Challenge: Odd Cycles (Blossoms)

  • In general graphs, odd-length cycles (blossoms) can “trap” potential augmenting paths.
  • Contract such blossoms into a single pseudo-vertex and continue searching for augmenting paths in the contracted graph.

🔹Blossom Expansion

  • When an augmenting path is found in the contracted graph, expand the blossoms back and reconstruct the real path in the original graph.

🧩 Code Structure

struct Graph_Blossom

Encapsulates all graph and algorithm operations.

Member Type Description
n, m int n = number of vertices; m = allocated space for blossoms (≈ 2n)
matched vector<int> Stores matched partner for each vertex (or -1 if unmatched)
depth vector<int> BFS layer parity: 0 = unvisited, 1 = even, 2 = odd
blsm vector<int> Current blossom representative for each node
parent vector<int> Parent pointers for BFS tree
adj_mat vector<vector<int>> Adjacency matrix representation
blossom_content vector<vector<int>> For each blossom, stores contained vertices

Key Functions

🔸 add_edg(u, v)

Adds an edge between u and v.

🔸 matcher(u, v)

Marks vertices u and v as matched (removes from adjacency).

🔸 tracer(x)

Traces path from vertex x to the BFS root following parent pointers, skipping inner blossom members.

🔸 contract_to_blsm(id, x, y, route_x, route_y)

Contracts a detected blossom into a single vertex (id), updates adjacency and blossom membership.

🔸 lift_path(route_x)

Reconstructs the original augmenting path from the contracted representation by expanding blossoms.

🔸 solve()

Main driver of the algorithm:

  1. Repeatedly runs BFS to find augmenting paths.
  2. Performs blossom contraction when needed.
  3. Augments the matching.
  4. Terminates when no augmenting path is found.

🕒 Time Complexity Analysis

Phase Description Complexity
BFS search Finding augmenting paths ( O(N^2) ) per phase
Blossom contraction/expansion Maintaining structure ( O(N^2) )
Number of phases Each augmentation increases matching size ( O(N) )
Total Edmonds’ classic bound O(N³)

🧩 Notes & Limitations

  • Designed for dense general graphs — adjacency matrix used.
  • Handles odd cycles through explicit blossom contraction.
  • Works for disconnected graphs.
  • Matching is maximum cardinality, not necessarily maximum weight (for weighted graphs, use Blossom V).
