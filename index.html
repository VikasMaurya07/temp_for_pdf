Vertex Cover Problem On Bipartite Graphs

In this project, we study the NP-hard Vertex Cover problem, focusing on a special and tractable class of graphs: bipartite graphs. We implement the fundamental algorithms designed specifically for this case and build a framework that can be applied to real-world scenarios; most notably, computing the vertex cover of a bipartite graph and solving a practical application: determining the minimum number of charts required to display the price movements of multiple stocks.

This report is organized as follows: we first present the theoretical background, then discuss the key theorems and algorithms, and finally develop and apply the code to these real-world problems. Click the link below for GitHub repo.

Given a graph G=(V,E)G = (V, E)G=(V,E);

‚àÄ(u,v)‚ààE,u‚ààC or v‚ààC\forall (u, v) \in E,\quad u \in C \text{ or } v \in C‚àÄ(u,v)‚ààE,u‚ààC or v‚ààC

  • C=[B]C = [B]C=[B] (covers both edges)
  • C=[A,C]C = [A,C]C=[A,C] (also covers both edges)
  • C=[A,B,C]C = [A,B,C]C=[A,B,C] (but not minimal)

In any finite bipartite graph G=(X‚à™Y,E)G=(X\cup Y, E)G=(X‚à™Y,E) the size of a maximum matching equals the size of a minimum vertex cover:
œÑ(G)=ŒΩ(G)\tau(G)=\nu(G)œÑ(G)=ŒΩ(G)
Where œÑ(G)\tau(G)œÑ(G) is the maximum number of pairwise disjoint edges (maximum matching size) and ŒΩ(G)\nu(G)ŒΩ(G) is the minimum number of vertices meeting every edge (minimum vertex cover size). We prove œÑ(G)‚â§ŒΩ(G)\tau(G)\le\nu(G)œÑ(G)‚â§ŒΩ(G) and then show ŒΩ(G)‚â§œÑ(G)\nu(G)\le\tau(G)ŒΩ(G)‚â§œÑ(G) by constructing a vertex cover of size equal to a maximum matching.

Easy direction:

Every edge in a matching must be covered by distinct vertices of any vertex cover (no two matched edges share an endpoint in a matching). Thus a vertex cover must contain at least one endpoint from each matched edge; so the cover size is at least the matching size. Hence
œÑ(G)‚â§ŒΩ(G)\tau(G)\le\nu(G)œÑ(G)‚â§ŒΩ(G)

Constructing a cover of size œÑ(G)\tau(G)œÑ(G):

Let MMM be a maximum matching in the bipartite graph G=(X‚à™Y,E)G=(X\cup Y,E)G=(X‚à™Y,E). We will build a vertex cover CCC with ‚à£C‚à£=‚à£M‚à£|C|=|M|‚à£C‚à£=‚à£M‚à£, proving ŒΩ(G)‚â§‚à£M‚à£=œÑ(G)\nu(G)\le|M|=\tau(G)ŒΩ(G)‚â§‚à£M‚à£=œÑ(G).

Build the alternating-reachability set:

  • Call a vertex free if it is not incident to any edge of MMM.
  • Consider the directed exploration that starts from all free vertices in XXX and follows alternating paths:
    • From a vertex in XXX follow edges not in MMM to YYY.
    • From a vertex in YYY follow edges in MMM to XXX.
  • Let ZZZ be the set of vertices reachable from free vertices of XXX by such alternating paths. Partition ZZZ as ZX=Z‚à©XZ_X=Z\cap XZX‚Äã=Z‚à©X and ZY=Z‚à©YZ_Y=Z\cap YZY‚Äã=Z‚à©Y.

Intuition: ZZZ is exactly the set of vertices reachable by alternating paths starting at unmatched left-side vertices.

Define the candidate cover:

C=(X‚àñZX) ‚à™ (ZY).C = (X\setminus Z_X)\ \cup\ (Z_Y).C=(X‚àñZX‚Äã) ‚à™ (ZY‚Äã).

In words: take all vertices of XXX that were not reached, together with all vertices of YYY that were reached.
We will show; CCC is a vertex cover and ‚à£C‚à£=‚à£M‚à£|C|=|M|‚à£C‚à£=‚à£M‚à£.

  • If u‚ààZXu\in Z_Xu‚ààZX‚Äã then by the definition of ZZZ every edge uwu wuw not in MMM that is outgoing from uuu to YYY leads to a reachable w‚ààZYw\in Z_Yw‚ààZY‚Äã. In particular, if uv‚ààEuv\in Euv‚ààE and uv‚àâMuv\notin Muv‚àà/M, then v‚ààZY‚äÜCv\in Z_Y\subseteq Cv‚ààZY‚Äã‚äÜC.
  • If uv‚ààMuv\in Muv‚ààM, and u‚ààZXu\in Z_Xu‚ààZX‚Äã, then the matched edge from uuu goes to some vvv; but our alternating reachability rules ensure that when a vertex in XXX is reachable, the matched edge ifanyif anyifany goes to the corresponding YYY vertex only if that YYY vertex is also reachable; so again v‚ààZY‚äÜCv\in Z_Y\subseteq Cv‚ààZY‚Äã‚äÜC.
  • If u‚àâZXu\notin Z_Xu‚àà/ZX‚Äã, then u‚ààX‚àñZX‚äÜCu\in X\setminus Z_X\subseteq Cu‚ààX‚àñZX‚Äã‚äÜC.

A compact way to see this: if an edge has its XXX-endpoint reachable then its YYY-endpoint is reachable too (by alternating-step rules) and so in CCC; otherwise the XXX-endpoint itself is in CCC.

Proving ‚à£C‚à£=‚à£M‚à£|C|=|M|‚à£C‚à£=‚à£M‚à£:

We count how many vertices of CCC come from XXX and from YYY.

Claim A. Every vertex of ZYZ_YZY‚Äã is matched (incident to an MMM-edge).
Proof: If y‚ààZYy\in Z_Yy‚ààZY‚Äã is reachable by an alternating path, the path alternates and the last step to yyy came along a non-matching edge from some x‚ààZXx\in Z_Xx‚ààZX‚Äã; but by the alternating rules from yyy we follow the matching-edge (if any) to continue. Kuhn‚Äôs Algorithm:

Kuhn‚Äôs algorithm is a DFS-based augmenting path algorithm for maximum bipartite matching.

  1. For each left vertex uuu, try to find an augmenting path using DFS.
  2. If an unmatched right vertex is found, or we can re-route an existing match, we match uuu.
  3. Each DFS either:
    • increases the matching size by 1, or
    • returns false.

Worst case: O(V √ó E)

Minimum Vertex Cover Using K√∂nig‚Äôs Theorem:

Given a maximum matching MMM, the minimum vertex cover in bipartite graphs is computed by:

  1. Mark all unmatched left vertices.
  2. Run DFS/BFS alternating between:
    • non-matching edges (L ‚Üí R)
    • matching edges (R ‚Üí L)
  3. Let:
    • visL = visited left vertices
    • visR = visited right vertices
  4. The minimum vertex cover is:
(Left \ visL)  ‚à™  (Right ‚à© visR)

Total Complexity:
Building adjacency: O(E)
DFS alternation: O(V + E)
Total: O(V √ó E) (due to matching step dominating)

    A=(A1,A2,‚Ä¶,Ak),B=(B1,B2,‚Ä¶,Bk)A = (A_1, A_2, \dots, A_k), \quad B = (B_1, B_2, \dots, B_k)A=(A1‚Äã,A2‚Äã,‚Ä¶,Ak‚Äã),B=(B1‚Äã,B2‚Äã,‚Ä¶,Bk‚Äã)

It is a direct instance of a very important and widely applicable combinatorial problem:

Minimum Path Cover in a DAG ‚Üî Maximum Matching in a Bipartite Graph
Minimum Path Cover: Smallest possible number of paths needed to cover all vertices without breaking the direction constraints.

This problem can be extended to a range of real world problems such as;

  • Task Scheduling with Precedence Constraints
  • Exam or Class Timetabling
  • Layering and Ranking in Graph Drawing
  • Comparative Genomics / DNA Sequence Evolution

Graph-Theoretic Reformulation:

  1. Create a directed edge A‚ÜíBA \to BA‚ÜíB if stock AAA is strictly lower than stock BBB at all time points.
  2. Convert this DAG into a bipartite graph by duplicating nodes into Left and Right partitions.
  3. Find the maximum bipartite matching, which corresponds to the maximum number of stock pairs that can be placed consecutively on a single chart.
  4. Using Dilworth‚Äôs theorem / minimum path cover in DAGs, the minimum number of charts required is:
    Minimum Charts=n‚àísize of maximum matching\text{Minimum Charts} = n - \text{size of maximum matching}Minimum Charts=n‚àísize of maximum matching

Dilworth's Theorem:

Dilworth‚Äôs Theorem is a fundamental result in order theory that connects two important concepts in a partially ordered set (poset):

  • Chains ‚Üí sequences where every pair of elements is comparable
  • Antichains ‚Üí sets where no two elements are comparable

In any poset, the minimum number of chains required to cover all elements equals the size of the largest antichain. In simpler terms: If many elements cannot be compared with each other (a large antichain exists), we need at least that many chains to cover the set.

In the Chart Minimisation problem:

  • Each stock is an element in a poset (defined by dominance of prices).
  • Each chart corresponds to a chain of comparable stocks.
  • The minimum number of charts equals the size of the largest antichain.

Using bipartite matching, we compute this via the equivalent formulation:
Minimum Path Cover=n‚àíMaximum Matching\text{Minimum Path Cover} = n - \text{Maximum Matching}Minimum Path Cover=n‚àíMaximum Matching

Algorithm for finding Chart Groupings:

Once the maximum bipartite matching has been computed (where each stock appears once on the left and once on the right), we convert the matching into actual chart groups. Each group corresponds to one directed chain of stocks that can be plotted together on the same chart. The algorithm consists of three main steps:

1. Build Successor and Predecessor Maps:
From the matching result:

  • If left-stock uuu is matched to right-stock vvv, we create a successor link u‚Üívu \rightarrow vu‚Üív
  • At the same time, we mark uuu as the predecessor of vvv.

This produces two arrays:

  • succ[u] = v ‚Üí the next stock in the chain
  • pred[v] = u ‚Üí the previous stock in the chain
    All unmatched stocks simply have succ = -1 and pred = -1. These links represent how stocks can be arranged sequentially in a valid chart.

2. Identify Chain Starts and Build Chains
A chain must start at a stock with no predecessor. For every such stock:

  • Start walking forward using succ
  • Collect all stocks along the path
  • Stop when no further successor exists or when a repeated node would introduce a cycle

Each walk yields one complete chart group. This constructs all ‚Äúmaximal‚Äù chains corresponding to maximum matching.

Building adjacency: O(E)

Total: O(V √ó E) (due to matching step dominating)
